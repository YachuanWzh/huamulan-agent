# SBS 评审队列删除设计

## 目标

让用户可以从智能体工程的 SBS 评审队列中删除待处理或已评审任务，并确保界面状态与持久化数据一致。

## 交互

- 每条队列记录保留原有的详情选择操作，并在右侧提供独立的“删除”按钮。
- 点击删除后使用浏览器确认框显示任务提示词；用户取消时不发送请求。
- 删除成功后刷新队列。如果删除的是当前选中任务，同时清空右侧详情。
- 删除失败时沿用工作台现有错误提示，不提前移除队列记录。

## 后端与数据

- 新增 `DELETE /api/sbs/tasks/{task_id}`。
- 任务存在时调用持久化层删除并返回 `204 No Content`；不存在时返回 `404`。
- `sbs_reviews.task_id` 已配置 `ON DELETE CASCADE`，因此删除任务会同时删除关联评审记录。
- 待处理和已评审任务均允许删除，不增加软删除、恢复站或权限模型。

## 前端结构

- 前端 API 客户端增加 `deleteSBSTask(taskId)`。
- 队列条目改为一行容器：主按钮负责打开任务，删除按钮只负责确认与删除，两个操作互不嵌套。
- 删除按钮提供包含任务名称的无障碍名称，避免图标按钮语义不明。

## 验证

- 后端测试覆盖成功删除和不存在时返回 `404`。
- API 客户端测试覆盖 `DELETE` 方法与路径。
- 组件测试覆盖取消删除、确认删除、刷新队列和清空当前详情。
- 运行 SBS 后端测试、前端相关测试、前端构建与静态检查。
